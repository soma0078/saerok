# PRD — 새록 (Saerok)

> 저장한 문장을 정해진 시간에 다시 상기시켜주는 알림 앱

## 1. 목적 및 배경

사용자가 마음에 담아둔 문장(말씀, 명언, 다짐 등)을 카테고리별로 저장하고, 설정한 시간에 알림으로 반복 노출하여 내면화를 돕는다.

## 2. 지원 환경

| 항목       | 내용                            |
| ---------- | ------------------------------- |
| 플랫폼     | iOS / Android                   |
| 프레임워크 | Expo (React Native, TypeScript) |
| 배포       | App Store / Google Play         |

## 3. 타겟 사용자

- 말씀/명언/다짐을 꾸준히 묵상하고 싶은 사용자
- 알림을 통해 잊지 않고 반복적으로 접하고 싶은 사용자

## 4. 핵심 기능 요구사항

### F-AUTH: 인증

| ID        | 요구사항                                          | 우선순위 |
| --------- | ------------------------------------------------- | -------- |
| F-AUTH-01 | 구글, 카카오 소셜 로그인                          | P2       |
| F-AUTH-02 | 로그인 성공 후 User 정보 AsyncStorage에 로컬 저장 | P2       |

### F-ONB: 온보딩

| ID       | 요구사항                                                              | 우선순위 |
| -------- | --------------------------------------------------------------------- | -------- |
| F-ONB-01 | 온보딩 소개 슬라이드 (앱 가치 소개, O-01~O-02)                        | P0       |
| F-ONB-02 | 알림 권한 요청 — 알림 설정 화면 진입 시 contextual 요청               | P0       |
| F-ONB-03 | 닉네임/사진 프로필 등록 (건너뛰기 가능) — Phase 2로 이전              | P2       |

### F-M: 문장 관리

| ID     | 요구사항                                              | 우선순위 |

| F-M-01 | 문장 구절 + 카테고리 입력 후 AsyncStorage에 로컬 저장 | P0       |

### F-A: 알림 스케줄링

| ID     | 요구사항                                             | 우선순위 |

| F-A-03 | 설정 조건 기반 로컬 알림 예약 (`expo-notifications`) | P0       |
| F-A-04 | 알림 메시지에 랜덤 또는 고정 문장 표시               | P0       |

## 5. 비기능 요구사항

| 항목     | 요건                                                         |
| -------- | ------------------------------------------------------------ |
| 서버     | Phase 1: 없음. 모든 데이터는 디바이스 AsyncStorage에 로컬 저장. Phase 2 이후 서버 도입 예정 |
| 알림     | `expo-notifications` 로컬 알림. 인터넷 불필요                                               |
| 오프라인 | 알림 스케줄링 포함 전체 기능 오프라인 동작                                                  |
| 권한     | 알림 권한 없이는 핵심 기능 사용 불가 → 온보딩 필수 단계 처리 |

## 6. 엣지 케이스 및 예외 처리

| 상황                     | 처리 방식                                         |
| ------------------------ | ------------------------------------------------- |
| 선택 카테고리에 문장 0개 | 해당 카테고리 알림 예약 제외 + Toast 안내         |
| 알림 권한 거부           | 설정 화면으로 유도하는 가이드 메시지 노출         |
| 기간 만료 (7일/30일)     | 앱 진입 시 재설정 유도 배너/팝업 노출             |
| 프로필 미등록(건너뛰기)  | 닉네임 기본값으로 대체, 설정에서 언제든 등록 가능 |

## 7. 범위 외 (Out of Scope) — Phase 1 기준

- 소셜 공유 기능
- 묵상 내용 입력 필드
- 웹/PWA 배포

## 8. 미래 계획 (Phase 2+)

| 항목 | 내용 |
| ---- | ---- |
| 서버 도입 | 사용자 데이터(문장, 카테고리, 알림 설정)를 서버에 저장 |
| 클라우드 동기화 | 로그인 계정 기준으로 디바이스 간 데이터 동기화 |
| 데이터 마이그레이션 | Phase 1 로컬 데이터를 서버로 이전 (User ID 키 기반) |

> 소셜 로그인(F-AUTH)은 Phase 2 서버 연동 시 계정 식별자로 활용하기 위해 Phase 1에서 미리 구현.

### F-AI: AI 명언 추천 (Phase 2+)

온보딩 시 사용자 신분을 입력받아 AI가 맞춤 명언을 추천하고 앱에 자동 등록하는 기능.

| ID      | 요구사항                                                                 | 우선순위 |
| ------- | ------------------------------------------------------------------------ | -------- |
| F-AI-01 | 온보딩에서 사용자 신분 선택 (직장인 / 취준생 / 학생 / 기타)              | P2       |
| F-AI-02 | 신분 기반으로 AI가 명언 추천 목록 생성 (LLM API 활용)                    | P2       |
| F-AI-03 | 추천 모드 선택 — 수동(목록에서 직접 선택) / 자동(N일에 M개 자동 등록)    | P2       |
| F-AI-04 | 추천 명언을 카테고리와 함께 문장 목록에 자동 등록                        | P2       |

**설계 제약**
- AI 추천은 인터넷 연결이 필요한 온라인 전용 기능 (초기 세팅 시 1회 호출)
- LLM 직접 생성 방식 사용 — 크롤링은 저작권/안정성 리스크로 제외
- 구체적인 API 선택은 ADR-003에서 결정
